冲突案例

开发人员A 开发需求1
开发了一个工具类 MathUtil
里面第一行写了一个方法 add(int [] args)；
同时开发人员B 开发需求2
开发了一个工具类 MathUtil
里面第一行写了一个方法 add(int a int b)。
他们在互相不知道对方需求的情况下、
同时提交了代码到自己的分支。
如果把需求1 和需求2 同时合并到主分支，
解决主分支MathUtil的冲突。

在r1和r2分支上分别创建文件MathUtil

将r1和r2两个分支合并到master主分支上后，MathUtil变成了冲突文件
